  > 其中，logback依赖已经因为依赖传递被引入了，不用再引入

  * **引入配置文件logback.xml**(我存在Desk里面了)（配置文件放resources目录下）

  * 定义一个日志记录器(日志对象)（用日志工厂get一个对象），
    然后用这个对象就可以调用方法(debug/info/...)来记录日志

  ```
  public class LogTest{
  	public static void main(String[arg]){
  		private static final Loggerlog = LoggerFactory.getLogger(LogTest.class);//这里的Logger是从org.slf4j包导进来
  	log.debug("开始计算。。。");
  		log.info("计算结果为：");
  	}
  }
  
  
  
  
  
  ```

  ```
  private static final Loggerlog = LoggerFactory.getLogger(LogTest.class);
  ```

  **这句代码基本固定，唯一要变的就是字节码对象XXX.class（在哪个类中使用日志，就用哪个类名）**

  > **也可以使用注解代替这个日志对象的声明**

## sql日志



* 使用log-impl: org.apache.ibatis.logging.stdout.StdOutImpl将sql语句打到控制台

  ```
  mybatis:
    #mapper配置文件
    mapper-locations: classpath:mapper/*.xml
    type-aliases-package: top.hazenix.entity
    configuration:
      #开启驼峰命名
      map-underscore-to-camel-case: true
      log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
  ```

  

nested exception is org.apache.ibatis.reflection.ReflectionException: There is no getter for property named 'query' in 'class com.zeroone.star.project.query.j8.finance.tradeinvoice.TradeInvoiceQuery

* 使用场景
  * 开发阶段 调试时  需要看具体的sql执行情况和具体执行的sql语句的时候
  * 平时关掉【占用空间过大，不方便看其他日志】
