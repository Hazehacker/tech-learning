# 批量插入优化方案





* 方案一：一次插入一个
  性能极差
  每条数据一次网络请求

  ```
  void testSaveOneByOne(){
  	Long b = System.currentTimeMillis();
  	for(int i = 1;i<=100000;i++){
  		userService.save(buildUser(i));
  	}
  	Long e = System.currentTimeMillis();
  	System.out.println("耗时：" + (e-b));
  	
  }
  ```

* 方案二：

  性能提升接近十倍，但是仍然很慢
  每1000条数据一次网络请求、**通过减少网络请求的次数**

  ```
  void testSaveBatch(){
  	
  	//装备一个容量为1000的集合
  	List<User> list = new ArrayList<>(1000);
  	long b = System.currentTimeMillis();
  	for(int i = 1;i<=100000;i++){
  		//添加一个user
  		list.add(buildUser(i));
  		//每1000条批量插入一次
  		if(i % 1000 == 0){
  			userService.saveBatch(list);
  			list.clear();
  		}
  	}
  	Long e = System.currentTimeMillis();
  	System.out.println("耗时：" + (e-b));
  }
  ```

  > 为什么一次性只插入1000条数据
  >
  > 1. 避免在内存中占用太多数据
  > 2. 向数据库传递数据的时候 数据请求的数据包是有一个上限大小的(一次网络请求的数据量是有限的)，如果一次性插入数据过多  传不过去

* 方案三：mybatis的foreach拼接、
  性能好

  所有数据一次批处理

  ```
  void testSaveBatch(){
  	
  	//装备一个容量为1000的集合
  	List<User> list = new ArrayList<>(1000);
  	long b = System.currentTimeMillis();
  	for(int i = 1;i<=100000;i++){
  		//添加一个user
  		list.add(buildUser(i));
  		//每1000条批量插入一次
  		if(i % 1000 == 0){
  			userService.saveBatch(list);
  			list.clear();
  		}
  	}
  	Long e = System.currentTimeMillis();
  	System.out.println("耗时：" + (e-b));
  }
  ```
