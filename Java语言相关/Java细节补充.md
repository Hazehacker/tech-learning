### **Java 中的“引用”**

> #### **地址 (Address)**
>
> **地址是对象在内存中的位置编号。**
>
> 就像每栋房子都有一个具体的街道地址（XX路XX号）一样，内存中的每个对象（或变量）都有一个唯一的编号，叫做**内存地址**。
>
> - 在C/C++中，你可以使用取地址符 `&` 来获取一个对象的地址。
> - **作用**：有了地址，程序才能准确地找到并读写这个对象。
>
> #### 引用 (Reference)**
>
> **引用是对象的“别名”或“外号”。**
>
> 这是C++等语言中的一个重要概念。引用并不是一个新的对象，它只是给已存在的对象起了一个**同义词**。
>
> - 特性：
>   - **必须初始化**：声明引用时，必须告诉它是谁的“外号”。
>   - **终身制**：一旦确定了是某个对象的引用，就不能再改指向别的对象了。
>   - **操作即本体**：你操作引用，就等同于直接操作原对象，不需要特殊的语法（不像指针需要解引用）
>
> #### **引用变量 (Reference Variable)**
>
> **引用变量是声明为引用类型的变量，它存储的是对象的引用（别名）。**
>
> 它是代码中用来承载“引用”这个概念的具体语法。在C++中，我们通常在变量名前加 `&` 来声明它。
>
> > ### **代码示例 (以 C++ 为例)**
> >
> > 假设我们有以下代码：
> >
> > ```cpp
> > int rate = 80;        // 1. 创建对象：在内存中创建一个名为 rate 的对象，值为 80
> > int& pt = rate;       // 2. 创建引用变量：pt 是 rate 的引用（别名），&在这里表示引用
> > ```
> >
> > - **对象**：`rate` 本身代表的那个内存块，里面存着数字 `80`。
> > - **地址**：`&rate`，这是 `rate` 在内存中的位置。
> > - **引用**：`pt` 就是 `rate` 的别名。此时 `pt` 和 `rate` 指向的是**同一块内存**。
> > - **操作**：如果你写 `pt = 100;`，实际上就是把 `rate` 的值改成了 `100`，因为 `pt` 就是 `rate` 的另一个名字

你可能会听到 Java 中也常说“引用”。在 Java 中，情况稍微有点不同：

- Java 中的**对象**都在堆内存里。
- 我们平时写的变量（如 `String str`），其实本质上是一个**引用变量**。
- 它内部存储的也是对象的**地址**（虽然 Java 不允许你直接打印或操作这个地址）。
- 所以在 Java 里，这个“引用变量”就像是**遥控器**，而“对象”是**电视机**。你通过遥控器（引用）来控制电视（对象），多个遥控器可以控制同一台电视。
